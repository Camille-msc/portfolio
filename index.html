Présentation générale de l'IA générative (2024–2025)

Introduction

Entre 2024 et 2025, l'intelligence artificielle générative s'impose comme une technologie centrale du numérique. Elle permet de produire du texte, du code, des images, de la voix ou des vidéos à partir de simples instructions humaines.

Des outils comme OpenAI ChatGPT, Google Gemini ou Anthropic Claude sont désormais intégrés au quotidien des particuliers comme des entreprises. L'IA générative n'est plus expérimentale : elle devient un outil de productivité standard, notamment dans le développement logiciel, la création de contenu et l'automatisation.

Cependant, cette adoption rapide soulève des enjeux importants : fiabilité des réponses, biais, sécurité des données et dépendance aux modèles propriétaires. Ces questions structurent aujourd'hui l'évolution de l'IA.

Retour à la veille

Ce que l'IA apporte aux humains (et aux développeurs)

Impact

L'IA générative agit comme un assistant cognitif. Elle ne remplace pas l'humain, mais augmente ses capacités.

Pour les humains en général :

  • Gain de temps sur les tâches répétitives ou complexes
  • Accès facilité à l'information et à la création
  • Démocratisation de compétences (écriture, design, analyse)

Pour les développeurs en particulier :

  • Aide à la génération de code, au debug et à la documentation
  • Accélération du prototypage et de l'apprentissage de nouveaux langages
  • Recentrage du métier sur la logique, l'architecture et la sécurité

Des outils comme GitHub Copilot ou Cursor transforment le développeur en superviseur intelligent de l'IA, capable de guider, corriger et valider les productions automatiques.

Retour à la veille

L'IA : avant, présent et futur

Évolution

Avant (avant 2022)

L'IA était surtout spécialisée et invisible : moteurs de recommandation, reconnaissance d'images, systèmes experts. Peu accessible au grand public.

Présent (2025 - 2026)

L'IA devient :

  • Générative (création de contenu)
  • Multimodale (texte, image, voix, code)
  • Interactive (dialogue naturel)

Elle est intégrée dans les IDE, les outils bureautiques et les plateformes cloud. La question n'est plus "peut-on l'utiliser ?" mais "comment bien l'utiliser ?"

Futur (2026)

Les tendances majeures :

  • IA agentique : capable de planifier et d'agir seule
  • Coopération entre plusieurs IA spécialisées (multi-agents)
  • Industrialisation de l'IA comme une infrastructure, au même titre que le cloud
  • Renforcement de la réglementation, notamment avec l'AI Act européen

L'humain restera central, mais son rôle évoluera vers la prise de décision, le contrôle et la responsabilité.

Application de réservation de DVD - RFTG

Application Full-Stack • BTS SIO

Présentation du projet

Système complet de gestion de DVD pour un parc à thème, incluant une interface web administrative et une application mobile pour les utilisateurs.

Architecture

Le projet RFTG est structuré en quatre composantes principales, chacune portant un nom inspiré de l'univers Mario :

  • Luigi - Application Mobile : Application Android développée en Java permettant aux clients du parc d'attraction de consulter et réserver des DVD depuis leur smartphone.
  • Mario - Application Web : Interface web d'administration développée en PHP permettant aux gestionnaires de gérer le catalogue de DVD, les réservations et les utilisateurs.
  • Toad - API Webservice : API REST sécurisée servant de pont entre les applications clientes (Luigi et Mario) et la base de données, garantissant une communication fluide et sécurisée.
  • Peach - Base de données : Base de données MySQL inspirée du modèle Sakila, stockant l'ensemble des informations relatives aux DVD, aux clients et aux réservations.

Détails du projet

  • Technologies utilisées : PHP, MySQL, Java, XML, REST API, Android Studio
  • Durée : Projet en cours
  • Modules : Gestion des réservations, authentification, interface mobile et web
  • Objectif : Créer un système complet de gestion de DVD pour un parc à thème

Défis techniques

Communication entre l'application mobile et le serveur via une API REST sécurisée, gestion multi-interface (web + mobile) et synchronisation des données avec la base MySQL.

MTCcongés - Application de gestion des congés

Application Java • BTS SIO

Présentation

Application de gestion des congés développée dans le cadre d'un projet BTS SIO SLAM. Conçue en Java 17 avec une interface Swing et une base de données MySQL, elle suit une architecture MVC structurée permettant une séparation claire entre logique métier, interface et persistance des données.

Objectifs du projet

Créer une solution complète permettant aux employés de demander des congés, aux responsables de les valider et aux administrateurs de gérer les comptes utilisateurs, le tout dans un environnement sécurisé et conforme aux bonnes pratiques.

Fonctionnalités principales

  • Authentification sécurisée : Connexion avec verrouillage automatique, réinitialisation de mot de passe, contrôles de session
  • Gestion des rôles : Admin (gestion utilisateurs), Responsable (validation demandes), Employé (dépôt et suivi)
  • Architecture MVC : Modèle (POJO + DAO), Vue (Swing adaptative), Contrôleur (logique métier)

Aspects techniques et sécurité

  • Protection contre injections SQL
  • Durée de session limitée
  • Vérifications côté client et serveur
  • Système de logging complet pour traçage et débogage
  • Conformité RGPD

Technologies utilisées

  • Langage : Java 17
  • Interface : Java Swing
  • Base de données : MySQL
  • Architecture : MVC avec DAO et POJO
  • Sécurité : Authentification, RGPD, logging

Compétences développées

Développement Java 17 & Swing • Architecture logicielle MVC • Conception de base de données MySQL • DAO & POJO • Sécurité, authentification, RGPD • Logging, débogage et gestion des erreurs • Gestion des rôles et permissions
